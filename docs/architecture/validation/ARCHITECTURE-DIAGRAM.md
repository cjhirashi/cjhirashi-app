## Summary

The architecture implements **defense-in-depth security** with multiple layers:

1. **Middleware**: Session validation and route protection
2. **RBAC**: Role-based access control on specific routes
3. **Database RLS**: Row-level security policies on tables
4. **JWT**: Token validation and expiry handling

Each layer works independently. If one layer is compromised, others still protect the system.
